### 🔐 Steps to Construct & Send Basic Auth Headers for Jira API

1. **Generate an API token**

   * Go to [Jira API Tokens](https://id.atlassian.com/manage/api-tokens)
   * Create a new token and copy it.

2. **Form the credentials string**

   * Format: `useremail:api_token`
   * Example: `johndoe@example.com:ABC123xyz456`

3. **BASE64 encode the string**

   * **Linux / Mac / Unix**: `echo -n "user@example.com:api_token_string" | base64`
   * **Windows PowerShell**:

     1. `$Text = "user@example.com:api_token_string"`
     2. `$Bytes = [System.Text.Encoding]::UTF8.GetBytes($Text)`
     3. `$EncodedText = [Convert]::ToBase64String($Bytes)`
     4. `$EncodedText`
   * Example output: `am9obmRvZUBleGFtcGxlLmNvbTpBQlMxMjN4eXo0NTY=`

4. **Create the Authorization header**

   * Format: `Authorization: Basic <BASE64_ENCODED_STRING>`
   * Example: `Authorization: Basic am9obmRvZUBleGFtcGxlLmNvbTpBQlMxMjN4eXo0NTY=`

5. **Send the request**

   * Example curl command:
     `curl -X GET -H "Authorization: Basic am9obmRvZUBleGFtcGxlLmNvbTpBQlMxMjN4eXo0NTY=" -H "Content-Type: application/json" "https://your-domain.atlassian.net/rest/api/2/issue/QA-31"`

1) Here our client server is "rahulshettyacademy.com" and Resource Authorization Server is "accounts.google.com", and resource owner is "Human" who is trying t access the website "https://rahulshettyacademy.com/getCourse.php"
2) In OAuth 2.0, the backend implementation of grant type means how your server (Authorization Server + Resource Server) handles different ways a client gets an access token. A grant type defines how the client proves identity and gets authorization.
3) Common grant types are:
   -> Authorization Code
   -> Client Credentials
   -> Password (legacy / mostly avoided now)
   -> Refresh Token
   -> Device Code (for TVs, IoT, etc.)
4) Scope : "scope" is nothing but application or client is asking what details it need from that owner, here owner is Human who wants to login into the client which is in our case "rahulshettyacademy.com"
5) Authorization Server URL : "auth_url" is a third party service like google, facebook etc. which will provide the identity verification for the "Human" who is trying t access the website "https://rahulshettyacademy.com/getCourse.php"
6) Client ID : "client_id" is a unique key or identity value, which will be generated when our client website "rahulshettyacademy.com" is first time registering with the third party identity verification provider services like "accounts.google.com", "facebook.com/dialog/oauth" etc.
7) Response Type : "response_type" is a unique one time password (OTP) or Code which we have to send for identity verification
8) Redirect URL/Callback URL : "redirect_uri" is a URL which will navigate back to the parent website or parent client which is in our case "rahulshettyacademy.com"
9) State : "state" is nothing but a unique encrypted token sent by the Human and should match with token registered with client server "rahulshettyacademy.com" and  Authorization Server "accounts.google.com", This is mainly used for security purpose to avoid An API hijacking attack.
10) Final Authorization URL to get the access code = Authorization Server URL + ? + scope=${Scope} + auth_url=${Authorization Server URL} + client_id=${Client ID} + response_type=${Response Type} + redirect_uri=${Redirect URL/Callback URL} + state=${State}
    Example: https://accounts.google.com/o/oauth2/v2/auth?scope=https://www.googleapis.com/auth/userinfo.email&auth_url=https://accounts.google.com/o/oauth2/v2/auth&client_id=692183103107-p0m7ent2hk7suguv4vq22hjcfhcr43pj.apps.googleusercontent.com&response_type=code&redirect_uri=https://rahulshettyacademy.com/getCourse.php
11) Now hit this above example 'Final Authorization URL' on your chrome browser and login with Google account. Once it is completed you will receive the Authorization Code in your browser URL but your web page will be a blank page.
12) Once you successfully logged in that OTP or Auth Code will be present in your browser URL, now copy that URL carefully and

14) Now after carefully copying the code or OTP our next step is to call actual Google Auth Server for Identity Verification of the Human who is trying to access the client Url "rahulshettyacademy.com"
15) Actual Resource Owner Endpoint or  Access Token URL is nothing but the Endpoint which fill generate the final access token after identity verification, in our case it is "https://www.googleapis.com/oauth2/v4/token"
16) Now we have to build the access token endpoint which is a POST request and it is concatenation of "Actual Resource Owner Endpoint + code=${Code} + client_id=${Client ID} + client_secret=${Client Secret} + redirect_uri=${Redirect URL/Callback URL} + grant_type=${GrantType}
    Example: https://www.googleapis.com/oauth2/v4/token?code=4%2F0Aci98E-kLK_vNF7-4jOPraQU_XOlME2odBr8DW2ZmubcImk25N_kvJuSTyz5AImpceadYA&client_id=692183103107-p0m7ent2hk7suguv4vq22hjcfhcr43pj.apps.googleusercontent.com&client_secret=erZOWM9g3UtwNRj340YYaK_W&redirect_uri=https://rahulshettyacademy.com/getCourse.php&grant_type=authorization_code#
17) Client Secret in OAuth 2.0 is a confidential password-like key used by the application (client) to prove its identity to the authorization server when requesting an access token.
